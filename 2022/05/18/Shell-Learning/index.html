一、Shell 简介

  • 什么是 Shell
    • Shell 是一个用C语言编写的程序，他是用户使用Linux的桥梁。Shell 既是一种命令语言，也是一种程序设计语言
    • Shell 是指一种应用程序，该应用程序提供了一个界面，用户通过该界面访问操作系统内核的服务
    • Ken Thompson 的 sh 是第一种 Unix Shell，Windows Explorer 是一个典型的图形界面shell
  • 概念区分：Shell 脚本
    • Shell 脚本(shell script)，是一种为shell编写的脚本程序
    • 业界所说的 shell 通常指 shell 脚本，但 shell 和 shell 脚本是两个不同的概念
    • 因此之后提到的 shell 编程 都是指 shell 脚本编程，不是开发 shell 本身
  • Shell 环境
    Shell 编程和其他脚本编程一样，需要的环境很简单：
    • 一个可以编写代码的文本编辑器
    • 一个能执行的脚本解释器

    Linux Shell 种类很多，常见的Shell

    • Bourne Shell (/usr/bin/sh或/bin/sh)
    • Broune Again Shell (/bin/bash)
    • C Shell (/usr/bin/csh)
    • K Shell (/usr/bin/ksh)
    • Shell for Root (/sbin/sh)
    • …..

    Tips:

    • 笔者使用的脚本解释器是Bash，即Broune Again Shell。Bash 是大多数Linux默认的Shell。
    • 笔者使用的文本编辑器是Vscode
    • 通常不区分 Bourne Shell 和 Broune Again Shell，因此 #!/bin/sh 同样可以改为 #!/bin/bash
    • #! 告诉系统其后路径路径的程序即是解释此脚本的 Shell 程序。

  • 变量类型
    运行 Shell 时，会同时存在三种变量
    • 局部变量：在脚本或命令中定义，仅在当前 shell 实例中有效，其他的 shell 启动的程序不能访问局部变量。
    • 环境变量：所有的程序，包括 shell 启动的程序，都能访问环境变量，有些程序需要环境变量来保证其正常运行。必要的 shell 脚本也可以定义环境变量。
    • shell 变量：shell 变量是 shell 程序设置的特殊变量。shell 变量有一部分是环境变量，有一部分是局部变量，这些变量保证了 shell 的正常运行。

十四、Shell 输入输出重定向

大部分Unix系统指令从终端接受输入并将产生的输出发送回你的终端。一个命令通常从一个叫标准输入的地方读取输入，将输出写入到标准输出，通常，这两个地方都是终端。
重定向命令列表：

命令 说明
command > file 将输出重定向至 file
command < file 将输入重定向至 file
command >> file 将输出以追加方式重定向至 file
n > file 将文件描述符为n的文件重定向至 file
n >> file 将文件描述符为n的文件以追加的方式重定向至 file
n >& m 将输出文件 m 和 n 合并
n <& m 将输入文件 m 和 n 合并
<< tag 将开始标记 tag 和结束标记 tag 之间的内容作为输入

  • 0 通常是标准输入 (STDIN)
  • 1 通常是标准输出 (STDOUT)
  • 2 通常是标准错误输出 (STDERR)

  • 深入理解重定向

    • 通常情况下，每个 Unix/Linux 命令运行时都会打开三个文件：
      • 标准输入文件(stdin): stdin 文件描述符为 0，Unix 程序默认从 stdin 读取数据
      • 标准输出文件(stdout): stdout 文件描述符为 1，Unix 程序默认向 stdout 输出数据
      • 标准错误文件(stderr): stderr 文件描述符为 2，Unix 程序向 stderr 流中写入错误信息

    Tips: 默认情况下，command < file 将 stdin 重定向到 file，command > file将 stdout 重定向到 file。

    • 重定向输入、输出与错误:
      • 合并 stdout 与 stderr: command > file 2>&1 或者 command >> file 1>&2
      • 对 stdin 与 stdout 都重定向: `command < file1 > file2
